/*
 * ===== 컬렉션 프레임워크 =====
 *
 * java.util 패키지에 포함된 자료구조 기반의 인터페이스 + 클래스 모음
 *
 * [ 주요 인터페이스 ]
 *
 * List  - 순서 O, 중복 O   ex) ArrayList, LinkedList, Stack, Vector
 * Set   - 순서 X, 중복 X   ex) HashSet, TreeSet
 * Queue - FIFO(선입선출)    ex) LinkedList, PriorityQueue
 * Map   - key+value 쌍     ex) HashMap, TreeMap, Hashtable, Properties
 *         (key 중복 X)
 *
 *
 * ===== List 컬렉션 =====
 *
 * 객체를 인덱스로 관리 → 저장 시 인덱스 부여, 인덱스로 검색/삭제 가능
 *
 * [ 주요 메서드 ]
 *
 * add(E e)                - 맨 끝에 추가
 * add(int index, E e)     - 해당 인덱스에 추가
 * set(int index, E e)     - 해당 인덱스 객체를 새 객체로 교체
 * contains(Object o)      - 저장 여부 확인 (boolean)
 * get(int index)          - 해당 인덱스 객체 리턴
 * isEmpty()               - 비어있는지 확인
 * size()                  - 전체 객체 수 리턴
 * clear()                 - 모든 객체 삭제
 * remove(int index)       - 해당 인덱스 객체 삭제
 * remove(Object o)        - 해당 객체 삭제
 *
 *
 * ===== ArrayList =====
 *
 * List<E> list = new ArrayList<E>();  // E 타입만 저장
 * List<E> list = new ArrayList<>();   // E 타입만 저장 (단축)
 * List    list = new ArrayList();     // 모든 타입 저장
 *
 * - 내부 배열에 객체 저장, 제한 없이 추가 가능
 * - 동일 객체 중복 저장 가능, null 저장 가능
 * - 인덱스 0번부터 차례대로 저장
 * - 중간 삽입/삭제 시 전체가 앞뒤로 1씩 밀림 → 빈번한 삽입/삭제에는 비적합
 * - 배열 크기 부족 시 자동으로 1.5~2배 늘리며 복사 → 성능 부하 유발 가능
 *   ex) 10 → 15 → 22 → 33 ...
 *   → 읽기/검색 전용으로 사용하는 것이 좋음
 * - RandomAccess 인터페이스 구현 (빠른 인덱스 접근을 명시적으로 표시)
 * - 자료 저장 방식: 동적 배열(Dynamic Array) - 연속적인 메모리 공간에 저장
 *
 *
 * ===== Vector =====
 *
 * List<E> list = new Vector<E>();  // E 타입만 저장
 * List<E> list = new Vector<>();   // E 타입만 저장 (단축)
 * List    list = new Vector();     // 모든 타입 저장
 *
 * - 동기화된 메서드로 구성 → 멀티 스레드가 동시에 Vector 메서드 실행 불가
 * - 멀티 스레드 환경에서 안전하게 객체 추가/삭제 가능 (thread-safe)
 *
 *
 * ===== LinkedList =====
 *
 * List<E> list = new LinkedList<E>();  // E 타입만 저장
 * List<E> list = new LinkedList<>();   // E 타입만 저장 (단축)
 * List    list = new LinkedList();     // 모든 타입 저장
 *
 * - 인접 객체를 체인처럼 연결해서 관리 (이중 연결 리스트, Doubly Linked List)
 * - 각 요소가 노드 객체로 존재, 이전/다음 노드를 가리키는 포인터 보유
 *   → 포인터 메모리 추가 사용
 * - 객체 삽입/삭제 시 앞뒤 연결만 바꾸면 됨 → 삽입/삭제가 빈번한 곳에서 유리
 * - 인덱스 접근 시 순차 탐색 필요 → 검색 느림
 * - Queue, Deque 특성도 함께 가짐 ex) poll(), offer() 등 지원
 *
 *
 * [ ArrayList vs LinkedList 비교 ]
 *
 *              ArrayList               LinkedList
 * 저장 방식    동적 배열                이중 연결 리스트
 * 인덱스 접근  빠름 (배열 기반)         느림 (순차 탐색)
 * 메모리 구조  연속적인 메모리 공간      노드 단위 분산 저장
 * 삽입/삭제    느림 (전체가 밀림)        빠름 (연결만 변경)
 * 적합한 상황  읽기/검색 위주           삽입/삭제 위주
 */
